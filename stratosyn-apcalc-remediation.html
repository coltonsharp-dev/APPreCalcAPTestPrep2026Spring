The polar graph shown below is a rose curve with 4 petals, each extending to a maximum radius of 2. Which equation defines this curve?

// TOPIC 3.14 · SKILL 2.B · CUMULATIVE GAP Q2
// FIG 1. POLAR ROSE CURVE — 4 PETALS, MAX RADIUS = 2
// WALKTHROUGH — WHY (B) IS CORRECT

For a rose curve r = a · sin(nθ) or r = a · cos(nθ):

• If n is even → 2n petals
• If n is odd → n petals
• The coefficient a = maximum radius

This graph has 4 petals with max radius 2. Using the rule: 4 petals = 2n → n = 2. So the equation is r = 2sin(2θ). The petals fall along the diagonal axes (π/4, 3π/4, 5π/4, 7π/4) which is consistent with sin(2θ).

// TRAP — WHY THE CLASS PICKED (A) AND (C)
(A) Limaçon confusion — r = 2 + 2cos(θ) creates a heart-shaped limaçon, not a rose. The class selected this because they saw a "symmetrical" shape and guessed. Key rule: if the equation adds a constant to a trig function, it's a limaçon. If it only has a trig function, it's a rose or circle.

(C) n=4 error — Students who knew the petal rule counted n=4 directly and wrote r=2sin(4θ). Remember: for even n, the number of petals is 2n — so you halve the petal count to find n.
// SELF-LABEL — WHAT WAS YOUR GAP?

G1
Concept gap — I didn't know the petal-count rule (even n → 2n petals, odd n → n petals).
G2
Curve-type confusion — I confused rose curves with limaçons and couldn't tell which equation family applied.
G3
Applied rule backward — I knew the rule but forgot to divide the petal count by 2 for even n, so I picked n=4 instead of n=2.
G4
Got it right — I applied the petal rule correctly, confirmed the max radius of 2, and selected (B).

On Exam 3 Q4, only 4 of 9 students found the correct x-intercept. The most common wrong answer was x = 4 — the vertical asymptote, not the intercept.

The function is defined by f(x) = log₂(x − 4) + 1. What is the x-intercept of the graph of f?

// TOPIC 2.13 · SKILL 1.A · CUMULATIVE GAP Q3
Find the x-intercept of f(x) = log₂(x − 4) + 1.
// WALKTHROUGH — WHY (C) IS CORRECT

The x-intercept is where f(x) = 0. Set up the equation:

log₂(x − 4) + 1 = 0
log₂(x − 4) = −1  ← subtract 1 from both sides
x − 4 = 2⁻¹ = 1/2  ← convert log to exponential form
x = 4.5

The key step is rewriting log₂(x−4) = −1 as 2⁻¹ = x−4. Don't forget the vertical shift (+1) — it changes the exponent from 1 to −1.

// TRAP — WHY MOST OF THE CLASS PICKED x = 4
Picking x = 4 means you found the vertical asymptote (where the log is undefined) instead of the x-intercept. These are different! The asymptote is where the input of the log = 0. The x-intercept is where the entire output of f = 0. Always set f(x) = 0 and solve — never just set the argument to zero.

(D) x = 6 — This happens when students ignore the +1 shift and solve log₂(x−4) = 0 instead of log₂(x−4) = −1. The +1 is critical to include before converting.
// SELF-LABEL — WHAT WAS YOUR GAP?
This error falls into one of two buckets. Which one was yours?
G1
Asymptote/intercept confusion — I set the argument (x−4) to zero and found the asymptote instead of the intercept.
G2
Dropped the vertical shift — I set log₂(x−4) = 0 instead of = −1, ignoring the +1 in the equation.
G3
Log-to-exponential error — I set up the equation correctly but converted the log incorrectly (e.g., thought 2⁻¹ = −2).
G4
Got it right — I set f(x)=0, subtracted 1, converted log to exponential form, and got x = 4.5.

// WALKTHROUGH — WHY (C) IS CORRECT

For f(x) = 3 · (1/2)ˣ, evaluate each characteristic independently:

y-intercept: Set x = 0. f(0) = 3 · (1/2)⁰ = 3 · 1 = 3. The y-intercept is (0, 3) — the coefficient, not the base.

(D) — Set the asymptote at y = 3 (the coefficient). The horizontal asymptote is always where the function approaches as x → ∞ — that is y = 0 for standard exponentials, not y = a.

On Exam 3 Q16, only 4 of 9 students computed the correct arc length. The class either forgot to include the radius or divided instead of multiplied.

An angle of 5π/6 radians is inscribed in a circle with radius 4 units. What is the arc length intercepted by this angle?

// TOPIC 3.2 · SKILL 3.B · CUMULATIVE GAP Q5
// WALKTHROUGH — WHY (C) IS CORRECT

The arc length formula is s = rθ, where θ is in radians and r is the radius.

s = 4 · (5π/6) = 20π/6 = 10π/3

This is the definition of radian measure: 1 radian = the angle that subtends an arc equal in length to the radius. So arc length = radius × angle in radians.

// TRAP — THE THREE WRONG MOVES
(A) Divided by r — Students wrote θ/r instead of rθ. This inverts the formula. Remember: s = rθ multiplies; it never divides.

(B) Reported the angle — Students gave back 5π/6 with no calculation. The angle and the arc length are equal only when r = 1 (unit circle). Always multiply by the actual radius.

(D) Doubled the radius — Students used r = 8 (maybe confused with diameter) or simplified 20π/6 incorrectly. Simplify carefully: 20π/6 = 10π/3, not 20π/3.
// SELF-LABEL — WHAT WAS YOUR GAP?
Arc length is one formula, but three things can go wrong. Which one?
G1
Formula inversion — I divided θ by r instead of multiplying rθ.
G2
Forgot to include radius — I reported the angle (5π/6) as the arc length without multiplying by r = 4.
G3
Arithmetic error — I set up s = rθ correctly but simplified 20π/6 wrong.
G4
Got it right — I applied s = rθ = 4(5π/6) = 10π/3 directly and correctly.

// TOPIC 2.6 · SKILL 2.A · CUMULATIVE GAP Q7

A good residual plot shows random scatter with no pattern around the zero line. When you see a pattern — especially a U-shape or curve — that means the linear model is systematically wrong.

Answer (D) says "consistent pattern" is a good thing. This is the exact backwards interpretation. A consistent pattern is bad — it means the model is missing something systematic. Random, patternless scatter is what you want. The word "consistent" sounds positive in English but signals a problem in statistics.

// TOPIC 1.4 · SKILL 2.A · CUMULATIVE GAP Q8
// FIG 3. GRAPH OF POLYNOMIAL f — IDENTIFY THE INFLECTION POINT
// WALKTHROUGH — WHY (B) IS CORRECT

A point of inflection is where the concavity of the graph changes — from concave up to concave down, or vice versa. It is not a maximum, minimum, or steepest point.

Looking at the graph: the left side curves upward (concave up — like a bowl). The right side curves downward (concave down — like a hill). The transition happens at x = 0. That is the point of inflection.

// TOPIC 2.10 · SKILL 2.A · CUMULATIVE GAP Q9
// WALKTHROUGH — WHY (B) IS CORRECT

To find the inverse of f(x) = e^(2x), swap x and y then solve for y:

x = e^(2y)

// TOPIC 4.1 · SKILL 1.A · CUMULATIVE GAP Q11 (UNIT 4 PCK)

To find the parameter value at a given point, solve both equations and verify both:

The first four terms of a sequence are 5, 15, 45, 135. Which expression correctly defines the nth term a(n) of this sequence?

// TOPIC 2.1 · SKILL 3.B · CUMULATIVE GAP Q12 (UNIT 2 PCK)
// WALKTHROUGH — WHY (B) IS CORRECT

First confirm the sequence type: 15/5 = 3, 45/15 = 3, 135/45 = 3. Constant ratio = 3 → geometric.

The standard formula for a geometric sequence is a(n) = a₁ · r^(n−1), where a₁ is the first term and r is the common ratio.

a₁ = 5, r = 3 → a(n) = 5 · 3^(n−1)

Verify: a(1) = 5·3⁰ = 5 ✓, a(2) = 5·3¹ = 15 ✓, a(3) = 5·3² = 45 ✓

(D) 3·5^(n−1) — The initial value (5) and common ratio (3) are swapped. Always identify: a₁ = first term in the sequence, r = ratio between consecutive terms. Don't guess which is which.
// SELF-LABEL — FINAL PHASE. BE PRECISE.

Model-type error — I identified the sequence as linear (picked constant difference instead of ratio) and chose a linear formula.

SKILL 3.C
Model Construction
Q1 · Q12 · Topics 1.13, 2.1
You see data growing and pick a model by feel — "it's increasing, so linear" or "it's curving, so quadratic." You skipped the systematic check: differences → ratios → model type. Same error in sequences: you guessed which number was the initial value instead of identifying a₁ = first term and r = consecutive ratio.
"The outputs were growing, so I picked quadratic — I never checked the ratio."
For every table problem: check in this order: (1) constant difference? → linear. (2) constant second difference? → quadratic. (3) constant ratio? → exponential. Do this mechanically every time. For sequences: always verify your formula by substituting n=1,2,3 before committing.
SKILL 2.B
Polar Representation
Q2 · Topic 3.13, 3.14
You couldn't match a polar graph to its equation because you didn't have the petal-count rule memorized, or you confused rose curves with limaçons. Both types look complex but obey exact rules. You were pattern-matching by appearance, not by formula structure.
"The graph had 4 petals so I put n=4 in the equation — I didn't know to halve it."
Memorize the two-rule cheat sheet: Rose r=a·sin(nθ): even n → 2n petals; odd n → n petals. Limaçon r=a+b·sin(θ): no petals, just a loop or dimple. Sketch one example of each type from memory until the shape is automatic.
SKILL 1.A
Log Intercept vs Asymptote
Q3 · Topic 2.13
You found the vertical asymptote when the question asked for the x-intercept — or you set f(x)=0 but forgot to account for the vertical shift before converting. These are the two exact mis-steps from Exam 3 Q4.
"I set x − 4 = 0 and said the intercept is x = 4. That's the asymptote."
Two-step fix: (1) Asymptote: set the LOG ARGUMENT to zero → solve for x. (2) x-intercept: set the entire function f(x) = 0, isolate the log, then convert to exponential form. These are different operations — never mix them up. Always identify which one the question is asking for first.
SKILL 3.A
Exp Function Anatomy
Q4 · Topic 2.4
You misread the coefficient as the base, labeled the asymptote at y = a (the coefficient) instead of y = 0, or called a decay function increasing because the coefficient looked large. Each error treats the coefficient and base as interchangeable — they are not.
"f(x) = 3·(1/2)ˣ — I thought the y-intercept was 1/2 because that's the base."
Three locked-in rules for f(x) = a·bˣ: (1) y-intercept = a (plug in x=0). (2) Growth if b>1, decay if 0<b<1. (3) Horizontal asymptote always y=0 for basic form. Write these three properties on a notecard and drill f(x) = 3·(1/2)ˣ, f(x) = 0.5·2ˣ, and f(x) = −4·eˣ from memory.
SKILL 1.C
Horizontal Shift in Tables
Q6 · Topic 1.12
You plugged x directly into the table (got f(3)) instead of applying the horizontal shift first (getting f(2)). Or you shifted in the wrong direction — looked up f(4) instead of f(2). The horizontal shift is the most consistently missed transformation in table problems.
"h(3) = f(3−1) + 3 — I just looked up f(3) and added 3. I didn't apply the shift."
Before looking anything up: substitute x into the full input expression first, then look up that value in the table, then apply the outer operation. For h(x) = f(x−1)+3, the lookup is ALWAYS f(x−1) — never f(x). Write the three-step process on scratch paper every time.
SKILL 2.A
Residual Plot Interpretation
Q7 · Topic 2.6
You read a patterned residual plot as evidence the model fits — because the residuals looked "consistent" or "small." This is the inverted interpretation: a pattern in residuals means the model is systematically wrong, not systematically right.
"The residuals made a clean curve, so I thought the model was good. That's backwards."
One rule, memorized: Random scatter around zero = model appropriate. Any visible pattern (U-shape, fan, curve) = wrong model type. When you see a residual plot, ask only: "Is this random scatter, or is there a shape?" Shape = red flag = consider a different model.
PROCESS
Misread the Question
Affects Q3, Q5, Q8, Q11
You solved a related but different problem: found the asymptote instead of the intercept, reported the angle instead of the arc length, identified an extremum instead of an inflection point, or found a parameter value that satisfies x(t) but not y(t).
"My calculation was right — I just solved for the wrong thing."
Before any calculation: circle the specific quantity being asked for, write it at the top of your scratch work, and check your final answer against it. "I found x = 4. Does x = 4 answer what was asked? No — the question asked for x-intercept, not asymptote." One check, every time.
CONCEPT
Definition Not Owned
Affects Q1, Q8, Q9, Q12
You didn't know the definition cold: what a point of inflection is (not an extremum), what the inverse procedure is (swap and solve), what the off-by-one in the geometric formula means (n−1 ensures a(1) = a₁). You guessed from partial knowledge.
"I knew there was a formula but I couldn't remember exactly how the pieces fit."
Build a one-page term sheet: every key definition from Units 1–4, stated precisely, with one example. For each definition, write: (1) the formal rule, (2) the most common wrong version, (3) how to tell them apart. Review it twice before May 11th.
